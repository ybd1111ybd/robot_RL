## 4. Action Graph 链路测试（当前主流程）

默认启用 gripper switch remap（本清单只保留这一种控制方式）：
- 只发布到 `/arm_left/joint_commands`、`/arm_right/joint_commands`。
- position 第 8 个值作为夹爪开合开关（`0=闭合，1=张开`）。
- 内部自动 remap 到 `*_mapped`，Action Graph 实际订阅 `*_mapped` 话题。

### 4.1 左臂命令

```bash
ros2 topic pub -r 20 /arm_left/joint_commands sensor_msgs/msg/JointState "{name: ['left_arm_joint1','left_arm_joint2','left_arm_joint3','left_arm_joint4','left_arm_joint5','left_arm_joint6','left_arm_joint7'], position: [2.00,2.00,0.00,-0.60,0.00,0.00,-0.00,1.0]}"
```

说明：当发送 8 个 position 且 name 只有 7 个关节时，第 8 个数会被当作 gripper switch（0=闭合，1=张开）。

### 4.2 左臂状态回读

```bash

```

预期：
- `name` 可能是整机关节（例如 19 个），这是 Action Graph 当前发布方式的正常现象。
- 左臂相关关节（`left_arm_joint1` 到 `left_arm_joint7`）的 `position` 会向目标值变化。

## 5. 常见问题与处理

- `ros2: command not found`：
  - 先执行 `source /opt/ros/humble/setup.bash`。
  - 推荐做第 1.1 节自动加载，后续免手动 `source`。
- 发布在刷屏但 Isaac 不动：
  - 先确认不是发了全 0（目标与当前位姿一致时看起来不动）。
  - 再确认关节名与当前 URDF 一致（本 URDF 使用 `left_arm_joint*` / `right_arm_joint*` / `body_joint*`）。
- 初始位姿能对上，但运行后被重力拉偏：
  - 提高保持力参数：`--hold-stiffness`、`--hold-damping`、`--hold-max-force`。
  - 先从 `--hold-stiffness 8000 --hold-damping 600 --hold-max-force 4000` 试起。
  - 若抖动明显，增大 `--hold-damping` 或降低 `--hold-stiffness`。
- 只有 `/parameter_events`、`/rosout`：
  - 先确认 Windows 端 Isaac 已启动并出现 `ROS2 Action Graph created.`。
  - 再确认 WSL 与 Windows 都是 `ROS_DOMAIN_ID=77` 且 `RMW_IMPLEMENTATION=rmw_fastrtps_cpp`。
  - 避免直接用 daemon 缓存，执行：
  - `ros2 daemon stop`
  - `ros2 topic list`
  - 若仍无话题，重新执行 `source /mnt/e/jz_robot/env.sh`，并确认 `~/.bashrc` 没有重复 source ROS2 脚本（见第 1.1 节）。
- `ros2: error: unrecognized arguments: --no-daemon`：
  - 当前环境的 ROS2 CLI 不支持该参数，直接使用 `ros2 topic list`、`ros2 topic hz`、`ros2 topic echo` 即可。
- RViz2 报 `No tf data` 或 `frame [base_link] does not exist`：
  - 重新执行 `source /mnt/e/jz_robot/env.sh`；
  - 确认 `fastdds_profile.xml` 和 `fastdds_profile_windows.xml` 都是 `<useBuiltinTransports>true</useBuiltinTransports>`；
  - 重新启动 Isaac 和 `isaac_rviz2_bridge.launch.py`。
- `numpy.dtype size changed` / `DLL load failed while importing _multiarray_umath`：
  - 确认 `run_with_isaac_fixed.bat` 已设置 `PYTHONNOUSERSITE=1`，避免加载 `C:\Users\<user>\AppData\Roaming\Python\Python311\site-packages` 中的 numpy。
  - 若仍报错，检查系统环境是否把 user-site 的 numpy 放进了 `PYTHONPATH` 或 `PATH`。
- `ros2 node list` 为空但话题存在：
  - 在 Isaac ActionGraph 链路里可能是正常现象，优先用 `ros2 topic info -v <topic>` 判断链路。
- `sequence size exceeds remaining buffer`：
  - 多与 DDS profile/网络配置不一致有关，优先重新执行：
  - `source /mnt/e/jz_robot/env.sh`
  - 并重启 Windows 端 `run_with_isaac_fixed.bat`。
- `URDF not found: ...`：
  - 优先显式传 `--urdf`（见第 2 节示例）。
  - 若仓库不在 `E:\jz_robot`，先在 PowerShell 设置 `$env:JZ_REPO_ROOT="你的仓库根目录"` 再启动。

## 7. 命令输入注意事项（避免误报接口错误）

- `sensor_msgs/msg/JointState` 不能手动拆成两行，否则会出现：
  - `The passed message type is invalid`
  - `JointState: command not found`
- 最稳妥做法是整条命令复制为一行执行。

错误示例（不要这样）：

```bash
ros2 topic pub ... sensor_msgs/msg/

```
